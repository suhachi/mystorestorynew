## 🔄 단계 1: Firebase Console에서 값 확인

### 1. Firebase Console 접속
- URL: https://console.firebase.google.com
- 프로젝트: `mystorestory`

### 2. 프로젝트 설정 열기
1. Firebase Console에서 프로젝트 선택
2. 좌측 메뉴에서 ⚙️ (설정) 아이콘 클릭
3. "프로젝트 설정" 선택

### 3. 웹 앱 설정 확인
1. "일반" 탭 선택
2. "내 앱" 섹션에서 웹 앱 선택 (없으면 추가)
3. "SDK 설정 및 구성" 섹션에서 다음 값 확인:
   - **apiKey**: `VITE_FIREBASE_API_KEY`
   - **authDomain**: `VITE_FIREBASE_AUTH_DOMAIN`
   - **projectId**: `VITE_FIREBASE_PROJECT_ID`
   - **storageBucket**: `VITE_FIREBASE_STORAGE_BUCKET`
   - **messagingSenderId**: `VITE_FIREBASE_MESSAGING_SENDER_ID`
   - **appId**: `VITE_FIREBASE_APP_ID`

### 4. 프로젝트 번호 확인
- "일반" 탭의 "프로젝트 번호"에서 확인
- 현재 값: `102904078280`

### 5. Google Analytics Measurement ID 확인 (선택)
1. "통합" 탭 선택
2. "Google Analytics" 섹션에서 확인
3. Measurement ID 형식: `G-XXXXXXXXXX`

## 🔄 단계 2: .env.local 파일 생성

### 1. 파일 생성
프로젝트 루트 디렉토리에 `.env.local` 파일을 생성합니다.

### 2. 내용 작성
`.env.local.example` 파일을 참고하여 다음 형식으로 작성:

```env
# Firebase API Key
VITE_FIREBASE_API_KEY=실제_API_키_값

# Firebase Auth Domain
VITE_FIREBASE_AUTH_DOMAIN=mystorestory.firebaseapp.com

# Firebase Project ID
VITE_FIREBASE_PROJECT_ID=mystorestory

# Firebase Storage Bucket
VITE_FIREBASE_STORAGE_BUCKET=mystorestory.appspot.com

# Firebase Messaging Sender ID
VITE_FIREBASE_MESSAGING_SENDER_ID=102904078280

# Firebase App ID
VITE_FIREBASE_APP_ID=실제_앱_ID_값

# Google Analytics Measurement ID (선택)
VITE_FIREBASE_MEASUREMENT_ID=G-XXXXXXXXXX
```

### 3. 파일 위치 확인
```
MY_STORE_STORYdesign/
├── .env.local          ← 여기에 생성
├── .env.local.example  ← 참고용
└── ...
```

## ✅ 검증 방법

### 1. 환경 변수 로드 확인
개발 서버 실행 시 환경 변수가 제대로 로드되는지 확인:

```bash
npm run dev
```

### 2. Firebase 초기화 확인
브라우저 콘솔에서 Firebase 초기화 확인:

```javascript
// 브라우저 콘솔에서 실행
console.log(import.meta.env.VITE_FIREBASE_PROJECT_ID);
// 예상 출력: "mystorestory"
```

### 3. 빌드 테스트
환경 변수가 빌드에 포함되는지 확인:

```bash
npm run build
```

## ⚠️ 주의사항

### 1. Git에 커밋하지 않기
- `.env.local` 파일은 **절대 Git에 커밋하지 마세요**
- 이미 `.gitignore`에 포함되어 있습니다
- 실제 API 키는 공개 저장소에 올리면 안 됩니다

### 2. 환경별 설정
- **로컬 개발**: `.env.local` 사용
- **스테이징**: Firebase Console에서 환경 변수 설정
- **프로덕션**: Firebase Console에서 환경 변수 설정

### 3. 보안
- API 키는 민감한 정보입니다
- 팀원과 공유할 때는 안전한 방법 사용
- 절대 공개 채널에 공유하지 마세요
