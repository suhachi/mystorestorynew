---

## 프롬프트

```
MyStoreStory의 사용자 인수 테스트를 수행합니다.

## 👥 User Acceptance Testing (UAT)

### 1. UAT 계획

#### 목표
- 실제 사용자 시나리오 검증
- 사용성 문제 발견
- 피드백 수집

#### 참가자
- **오너** (5명): 카페, 음식점 운영자
- **고객** (10명): 실제 주문 고객
- **관리자** (2명): 플랫폼 관리자

#### 기간
- **준비**: 1주
- **테스트**: 2주
- **분석**: 1주

---

### 2. 테스트 시나리오

#### Scenario 1: 앱 생성 (오너)

**목표**: 10분 내 앱 생성

**시나리오**:
1. 회원가입
2. 기본 정보 입력
3. 플랜 선택 (Pro)
4. 주문/결제 설정
5. 포인트 설정
6. 브랜딩 설정
7. 앱 생성 확인

**성공 기준**:
- [ ] 10분 내 완료
- [ ] 오류 없이 진행
- [ ] 만족도 4/5 이상

---

#### Scenario 2: 메뉴 등록 (오너)

**목표**: 메뉴 10개 등록

**시나리오**:
1. 대시보드 접속
2. 메뉴 관리 메뉴
3. 메뉴 추가 (이미지 포함)
4. 옵션 설정 (ICE/HOT, 사이즈)
5. 카테고리 분류

**성공 기준**:
- [ ] 메뉴당 2분 이내
- [ ] 이미지 업로드 성공
- [ ] 직관적인 UI

---

#### Scenario 3: 주문하기 (고객)

**목표**: 3분 내 주문 완료

**시나리오**:
1. 앱 접속
2. 메뉴 탐색
3. 메뉴 선택 및 옵션
4. 장바구니 추가
5. 주문하기
6. 정보 입력
7. 결제

**성공 기준**:
- [ ] 3분 내 완료
- [ ] 직관적인 플로우
- [ ] 에러 없음

---

### 5. 분석

#### 정량 데이터
| 시나리오 | 목표 시간 | 평균 시간 | 성공률 | 만족도 |
|----------|-----------|-----------|--------|--------|
| 앱 생성 | 10분 | 8.5분 | 100% | 4.6/5 |
| 메뉴 등록 | 20분 | 18분 | 95% | 4.4/5 |
| 주문하기 | 3분 | 2.8분 | 100% | 4.8/5 |

#### 정성 피드백

**긍정**:
- "정말 쉽고 빨랐어요!"
- "UI가 직관적입니다"

**부정**:
- "이미지 업로드가 느려요"
- "도움말이 더 필요해요"

---

### 6. 개선 계획

**High Priority**:
1. 이미지 업로드 속도 개선
2. 온보딩 튜토리얼 추가
3. 도움말 시스템 강화

**Medium Priority**:
1. UI 개선 (피드백 기반)
2. 에러 메시지 명확화
3. 로딩 시간 단축

**Low Priority**:
1. 추가 기능 요청
2. 디자인 미세 조정

---

### 7. 재테스트

**개선 후**:
- 2주 후 동일 시나리오 재테스트
- 개선 효과 측정
- 만족도 재조사

IMPORTANT: 실제 사용자 참여, 실 환경 테스트, 피드백 수집 및 반영
```
