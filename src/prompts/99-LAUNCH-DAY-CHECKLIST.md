### 비상 대응

#### Critical Issue (즉시 대응)

**서비스 다운**:
```
1. 즉시 Slack #emergency 채널 알림
2. 모든 개발자 소집
3. 롤백 준비
4. 원인 파악
5. 수정 또는 롤백 결정 (15분 내)
```

**롤백 절차**:
```bash
# 1. 이전 버전 확인
firebase hosting:releases

# 2. 롤백 실행
firebase hosting:rollback

# 3. 검증
npm run test:smoke

# 4. 공지
# Slack, 이메일, SNS
```

---

#### High Issue (1시간 내)

**에러율 급증**:
```
1. 에러 로그 확인
2. 영향 범위 파악
3. 임시 조치 (Feature Flag)
4. 근본 원인 분석
5. 수정 배포
```

**성능 저하**:
```
1. 성능 메트릭 확인
2. 병목 지점 파악
3. Scaling 조정
4. 최적화 적용
```
